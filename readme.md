# PomaVPN — Android-клиент

Android-клиент для собственного VPN-сервера на UDP с шифрованием
`xchacha20poly1305_ietf` через `libsodium`.

Серверная часть — отдельный проект на C++ (Linux).

---

## Возможности

- VPN-туннель поверх UDP
- Шифрование пакетов на `libsodium` (`xchacha20poly1305_ietf`)
- Работа как **foreground service** с уведомлением
- Сохранение настроек (`IP сервера`, `порт`, `IP клиента`, `ключ`, `MTU`)
- Отдельный экран с кнопками **Подключить / Отключить**
- Статус VPN: `Отключён`, `Подключается`, `Подключён`, `Ошибка`
- Счётчики **Отправлено / Получено** (обновление раз в секунду)
- Запрос разрешения на уведомления (Android 13+)
- Кнопка перехода в системные настройки уведомлений
- Сборка под `arm64-v8a`, `armeabi-v7a`, `x86`, `x86_64`

---


## Требования

- Android Studio (поддержка C++ / NDK)
- Android NDK (например `28.2.13676358`)
- CMake `3.22.1`
- Минимальный SDK: `24`
- Android 7.0 (Nougat) и выше
- Серверная часть VPN (отдельный проект)


## Сборка libsodium под Android

Перед сборкой Android-приложения нужно собрать `libsodium`
под все нужные архитектуры.

### 1. Подготовка

Распакуй исходники в `third_party/libsodium-1.0.20`.

### 2. Запуск скрипта

```bash
cd third_party
./build-libsodium-android.sh
```

Скрипт компилирует `libsodium` под 4 ABI:

- `arm64-v8a`
- `armeabi-v7a`
- `x86_64`
- `x86`

Результат сборки:

```text
third_party/libsodium-android/
  arm64-v8a/include/sodium.h
  arm64-v8a/lib/libsodium.a

  armeabi-v7a/include/sodium.h
  armeabi-v7a/lib/libsodium.a

  x86_64/include/sodium.h
  x86_64/lib/libsodium.a

  x86/include/sodium.h
  x86/lib/libsodium.a
```

## Совместимость

- Минимальный Android: 7.0 (API 24)
- Тестировалось на эмуляторе и реальном устройстве (`arm64-v8a`)
- Поддержка нескольких клиентов: каждый клиент должен иметь
  **уникальный IP внутри VPN-сети**, например:

```text
сервер:      192.168.200.1
ПК-клиент:   192.168.200.2
Android #1:  192.168.200.3
Android #2:  192.168.200.4
```

## Известные ограничения

- Ключ хранится в `SharedPreferences` в обычном виде.
- Нет автоматического переподключения при разрыве.
- Нет импорта/экспорта конфигурации.
- На некоторых устройствах при остановке возможна небольшая задержка
  на `read()` из TUN (можно решить переводом сокетов в `O_NONBLOCK`).
